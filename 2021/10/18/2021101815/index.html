index.php

首先
$func('', $this->arg); 可利用create_function()匿名函数代码注入，第二个参数存在RCE的风险

绕过preg_match(‘/^[a-z0-9]*$/isD’, $this->func)
/i不区分大小写
/s匹配任何不可见字符，包括空格、制表符、换页符等等，等价于[fnrtv]
/D如果使用$限制结尾字符,则不允许结尾有换行;
可以传入\create_function给$func来进行绕过

\create_function就是调用全局的create_function函数

xxc

起点Control\State\StopHook::__destruct()

令$this->processes[] = new MyGenerator();

触发Faker\MyGenerator::__get()，使得返回的$this->defaultValue=1

调用$process->stop();，触发Faker\MyGenerator::__call()

令$this->defaultCall=new GetFile(); 让echo $this->defaultCall;触发Method\Func\GetFile::__toString()

往下进入Method\Func\GetFile::getFiles()

令$this->flag = new GetDefault();

isset($this->flag->{$this->value})触发Method\Func\GetDefault::__isset()

进入Method\Func\GetDefault::popup()

$this->source = new GenerateFile(); 并使得$this->source->flag = 'myTest';来到return $s($length);

触发Method\Func\GenerateFile::__invoke()

令$this->source->generate = new SerializableClosure();在closure\src里

触发Opis\Closure\SerializableClosure::__invoke()可造成命令执行

call_user_func去触发SerializableClosure这个方法里面的__invoke，配合closure闭包类对象，其中封装的匿名函数功能是自定义的。

闭包是不可序列化的，尝试序列化闭包将导致异常，克服这个问题的解决方案是将闭包包装成一个Opis\Closure\SerializableClosure 对象，然后使用标准的serialize.
